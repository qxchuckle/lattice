# /lattice/spec/update/global

目标：把对多用户、多项目都成立的高层规则沉淀到全局 spec。

## 什么时候使用

- 这条规则已经不是某个项目或某个用户的偏好
- 它具有明显的通用性和稳定性
- 作为全局默认规则能减少重复沟通

## 执行步骤

1. 先确认这条规则足够通用：

```bash
lattice search "<相关主题>"
lattice spec list --scope global
```

2. 检查是否已经有类似的全局规则，优先补充而不是重复创建
3. 写入或更新全局 spec 文件

## 写入原则

- 全局 spec 只保留高价值、低争议、可迁移的规则
- 适合放全局的通常是：命名原则、文档结构、任务管理习惯、通用协作规则
- 不要把技术栈特定规则或团队私有约定写到全局

## 输出要求

- 说明为什么这条规则适合全局级
- 说明更新了哪个全局 spec 文件
- 提醒用户项目级和用户级规则仍然可以覆盖全局默认规则
