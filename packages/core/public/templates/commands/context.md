# /lattice/context

> **[执行前必读]** 执行本命令前，必须先使用 Skill 工具调用 `lattice` skill，阅读完整的 Lattice 使用说明，再继续执行后续步骤。

目标：快速获取当前项目或当前任务的高信号上下文，作为后续实现和分析的起点。

## 执行步骤

1. 如果当前目录属于已注册项目，先运行：

```bash
lattice context
lattice status
```

2. 如果用户在命令后提供了任务 ID，再额外运行：

```bash
lattice context --task <task-id>
```

3. **按当前请求/任务主题挑选并精读相关 spec**：spec 不只是约束规范，**也是 AI 认识项目的稳定经验**（项目结构、模块职责、领域概念、设计动机、方法论）。`lattice context` 输出的只是 spec 标题列表，**摘要常为"[缺失摘要]"，看标题不等于了解内容**。必须：

   - 对认知类 spec（系统架构 / 模块边界 / 领域概念 / 方法论 / 项目结构 / 关键流程地图）—— 默认 read_file 读正文
   - 对约束类 spec（编码规范、提交流程等）—— 按本次会修改的代码范围决定是否读
   - 若运行了 `lattice context --task <id>`，参考其输出的"语义关联 Spec"列表辅助挑选
   - 摘要缺失或过于笼统的 spec —— 必须读正文
   - 反模式：仅复述 spec 标题就开始作答 / 改代码

4. 总结输出时优先说明：

- 当前项目最重要的 spec 规则（已 read_file 精读后的提炼，而非标题罗列）
- 当前活跃任务及其状态
- 是否存在多层级 spec 冲突
- 是否有值得参考的关联项目

## 输出要求

- 不要直接转储整份上下文
- 优先提炼与当前用户请求最相关的规则和风险
- 如果当前目录不是 Lattice 项目，明确告诉用户，并建议先运行 `lattice link`
