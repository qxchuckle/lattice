# /lattice/task/archive

目标：结束并归档一个任务，同时提醒是否需要沉淀规范。

## 如何理解命令参数

- 命令后内容应视为任务 ID
- 如果命令后没有内容，或内容不是有效的任务 ID：先把这些信息视为当前任务线索，并结合当前对话总结出一个候选任务标题或主题；不要直接把原始命令参数当成任务标题。然后在当前项目中查找正在进行中的候选任务，再让用户二次确认是否归档

## 执行步骤

### 情况一：命令后有任务 ID

1. 先读取任务详情：

```bash
lattice task info <task-id>
```

2. 归档前，先更新一次该任务的 `prd.md`：

- 补充本次任务最终采用的方案
- 记录关键实现结果、取舍和遗留问题
- 增加一段“任务完成总结”，说明这次任务实际完成了什么
- 如果 `prd.md` 过长，可以把详细复盘、设计附录或交付清单拆到其他 Markdown 文件中渐进式加载；但 `prd.md` 仍然必须是该任务的必要入口，并保留摘要、索引和总结

3. 如果任务尚未完成，先将其标记为完成：

```bash
lattice task complete <task-id>
```

4. 再归档：

```bash
lattice task archive <task-id>
```

### 情况二：命令后没有内容，或内容不是任务 ID

如果当前目录是已注册项目，先运行：

```bash
lattice task list --current
lattice search "<根据当前对话和命令参数总结出的任务标题或主题>" --project <project-id> --type task
```

然后判断：

- 当前项目里哪些任务状态是 `in_progress`
- 命令参数如果不是任务 ID，只把它当作归纳任务主题的线索，而不是直接当作任务标题或任务 ID
- 哪个任务最像当前对话正在收尾的任务
- 如果有明显候选，先告诉用户你准备归档哪个任务，并请求用户二次确认
- 只有在用户确认后，才继续按“情况一”的步骤读取详情、完成并归档
- 如果没有匹配的可归档任务，也要明确告诉用户当前没有合适候选；同时可以提示：如果需要，我可以根据当前对话先新建一个任务，补上必要描述和完成总结后再帮你归档

如果当前目录不是已注册项目，明确提示无法自动推断当前项目任务。

## 归档前检查

在输出中补充判断：

- 本次任务是否形成了新的长期规则
- 是否应该更新项目 spec
- 是否有一部分经验应该提升为用户 spec 或全局 spec
- `prd.md` 是否已经补上最终方案和任务完成总结
- 如果 PRD 已拆分，`prd.md` 是否仍然保留为任务主入口

## 输出要求

- 告诉用户任务已归档
- 简短总结任务完成结果
- 如果命令参数为空，或命令参数不是任务 ID 且你推断出了候选任务，先向用户确认是否归档该任务，不要直接执行归档
- 如果没有匹配的可归档任务，明确告诉用户当前没有候选，并补充可以帮用户根据当前对话新建一个任务后再归档
- 归档前先更新 `prd.md`，补充任务的描述和完成总结
- 如果 PRD 已拆分，仍通过 `prd.md` 作为归档总结的主入口
- 如果发现应该沉淀规范，明确指出建议使用哪个 `/lattice/spec/update/*` 命令
